Updating Content

So, what just happened? The Cadmium CLI Commit command was just used to upload a directory of static content onto this website. Let's take a closer look at the command used to do this.

The Commit Command

The Cadmium CLI Commit command is the entry point for content being distributed by Cadmium. The command takes a directory of static content on the local file system and pushes it up to GitHub, so that it can be served by Cadmium sites. Here is the format of the command:

> cadmium commit -m <message> <dir> <site>

Although that is a short command, there is a lot going on:

  1. Content repository information is retrieved from site.
  2. The respository is cloned and its content is replaced with the content in dir.
  3. The changes are pushed back to GitHub with a commit message of message.
  4. site is then instructed to move to the new revision.

Once the development site has an acceptable version of the content, it is time to move that content to other environments. To do that, we need the clone command.
