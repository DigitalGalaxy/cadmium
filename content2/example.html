Commit Clone Example

Example Production Push

Let's look at an example production push. For this example, imagine the following environments are being used to push content to production:

  1. http://dev.cadmium.meltdemo.com - Development
  2. http://qa.cadmium.meltdemo.com - QA
  3. http://staging.cadmium.meltdemo.com - Staging
  4. http://prod.cadmium.meltdemo.com - Production

During development, we run this command to get local content onto the development site.

> cadmium commit -m "Some dev completed!" ./out http://dev.cadmium.meltdemo.com

Once we have a good version on the development server, we would run the following command to move that version into QA.

> cadmium clone -m "This stuff works!" http://dev.cadmium.meltdemo.com http://qa.cadmium.meltdemo.com

Now the content is ready to be tested. Assuming that everything looks good in testing, the content will then be moved to staging.

> cadmium clone -m "We know it works!" http://qa.cadmium.meltdemo.com http://staging.cadmium.meltdemo.com

The client can now review the content. When everyone agrees that the content is ready to launch, we clone it into production.

> cadmium clone -m "It is ready!" http://staging.cadmium.meltdemo.com http://prod.cadmium.meltdemo.com

Although the Cadmium CLI provides many commands, those are the only ones you should need to get content out to the public.
